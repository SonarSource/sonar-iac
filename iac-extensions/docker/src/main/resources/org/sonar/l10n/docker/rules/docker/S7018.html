In Dockerfiles, when using multi-line RUN instructions, itâ€™s important to ensure that the arguments are sorted alphabetically. This practice enhances the readability and maintainability of the code. It allows for easier tracking of modifications and can help prevent potential errors.

Why is this an issue?

When arguments in RUN instructions are not sorted alphabetically, it can make the Dockerfile harder to read and maintain. However, when arguments are sorted, it is easier to track changes in version control systems and to locate specific arguments.

Code examples

Noncompliant code example

FROM ubuntu:20.04

RUN apt-get update && apt-get install -y \
    unzip \
    wget \
    curl \
    git \
    zip

Compliant solution

FROM ubuntu:20.04

RUN apt-get update && apt-get install -y \
    curl \
    git \
    unzip \
    wget \
    zip

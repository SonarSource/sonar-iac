Azure Logic Apps store the inputs and outputs of every action in the workflow run history by default. If Secure Inputs and Secure Outputs are not enabled, sensitive payloads -- tokens, credentials, PII, API keys -- are logged in plain text and visible to anyone with access to the run history.

Why is this an issue?

Without enabling secureData properties for both "inputs" and "outputs", sensitive information can be exposed through:

  • Unintentional exposure of secrets and credentials in run history
  • Compliance violations (GDPR, ISO 27001, SOC 2)
  • Lateral movement opportunities if an attacker gains read access to run history

Noncompliant Code Example

{
  "type": "Microsoft.Logic/workflows",
  "apiVersion": "2019-05-01",
  "name": "no-secure-data",
  "properties": {
    "definition": {
      "actions": {
        "HTTP_Call": {
          "type": "Http",
          "inputs": {
            "method": "POST",
            "uri": "https://api.example.com/sensitive"
          }
        }
      },
      "triggers": {}
    }
  }
}

resource workflow 'Microsoft.Logic/workflows@2019-05-01' = {
  name: 'no-secure-data'
  location: resourceGroup().location
  properties: {
    definition: {
      actions: {
        HTTP_Call: { // Noncompliant
          type: 'Http'
          inputs: { method: 'POST', uri: 'https://api.example.com/sensitive' }
        }
      }
      triggers: {}
    }
  }
}

Compliant Solution

{
  "type": "Microsoft.Logic/workflows",
  "apiVersion": "2019-05-01",
  "name": "fully-secured",
  "properties": {
    "definition": {
      "actions": {
        "HTTP_Call": {
          "type": "Http",
          "inputs": {
            "method": "POST",
            "uri": "https://api.example.com/sensitive"
          },
          "runtimeConfiguration": {
            "secureData": {
              "properties": ["inputs", "outputs"]
            }
          }
        }
      },
      "triggers": {}
    }
  }
}

resource workflow 'Microsoft.Logic/workflows@2019-05-01' = {
  name: 'fully-secured'
  location: resourceGroup().location
  properties: {
    definition: {
      actions: {
        HTTP_Call: {
          type: 'Http'
          inputs: { method: 'POST', uri: 'https://api.example.com/sensitive' }
          runtimeConfiguration: {
            secureData: { properties: ['inputs', 'outputs'] }
          }
        }
      }
      triggers: {}
    }
  }
}

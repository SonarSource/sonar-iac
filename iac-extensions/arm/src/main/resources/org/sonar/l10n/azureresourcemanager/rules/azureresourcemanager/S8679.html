Azure Logic Apps rely on API connections (Microsoft.Web/connections) to authenticate against external services. When these connections are created using a personal or corporate user account, several risks arise.

Why is this an issue?

  • Access loss -- when the user leaves the organization or their account is disabled, all Logic Apps depending on that connection break silently
  • Credential sharing -- the user's credentials are embedded in a shared resource, violating the principle of least privilege
  • Audit and compliance gaps -- actions performed through the connection are attributed to an individual rather than a service identity

API connections should instead use service principals or managed identities.

Noncompliant Code Example

{
  "type": "Microsoft.Web/connections",
  "apiVersion": "2016-06-01",
  "name": "sql-connection",
  "properties": {
    "api": {
      "id": "[subscriptionResourceId('Microsoft.Web/locations/managedApis', resourceGroup().location, 'sql')]"
    },
    "parameterValues": {
      "server": "myserver.database.windows.net",
      "database": "mydb",
      "username": "john.doe@company.com",
      "password": "[parameters('sqlPassword')]"
    }
  }
}

resource sqlConnection 'Microsoft.Web/connections@2016-06-01' = {
  name: 'sql-connection'
  location: resourceGroup().location
  properties: {
    api: {
      id: subscriptionResourceId('Microsoft.Web/locations/managedApis', resourceGroup().location, 'sql')
    }
    parameterValues: {
      server: 'myserver.database.windows.net'
      database: 'mydb'
      username: 'john.doe@company.com' // Noncompliant
      password: sqlPassword
    }
  }
}

Compliant Solution

{
  "type": "Microsoft.Web/connections",
  "apiVersion": "2016-06-01",
  "name": "sql-connection-mi",
  "properties": {
    "api": {
      "id": "[subscriptionResourceId('Microsoft.Web/locations/managedApis', resourceGroup().location, 'sql')]"
    },
    "parameterValueType": "Alternative",
    "alternativeParameterValues": {}
  }
}

resource sqlConnectionMi 'Microsoft.Web/connections@2016-06-01' = {
  name: 'sql-connection-mi'
  location: resourceGroup().location
  properties: {
    api: {
      id: subscriptionResourceId('Microsoft.Web/locations/managedApis', resourceGroup().location, 'sql')
    }
    parameterValueType: 'Alternative'
    alternativeParameterValues: {}
  }
}
